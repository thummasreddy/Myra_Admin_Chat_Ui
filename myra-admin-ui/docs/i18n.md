# Translation Process

The app is currently English-first and includes a lightweight provider in `src/i18n/index.tsx` for language detection, switching, and typed translation keys. If localization grows beyond a few locales, migrate the provider to `react-i18next` and keep translation files under `src/i18n/locales`.

Recommended structure:

```text
src/i18n/
  index.ts
  locales/
    en/common.json
    es/common.json
```

Current process:

1. Add typed keys to `src/i18n/index.tsx`.
2. Replace hardcoded UI strings with `useI18n().t(...)` during feature work.
3. Keep validation messages in translation files or in a shared validation-message helper.
4. Add another locale to `supportedLocales`.
5. Add a language selector in account/settings once at least two locales are supported.
6. Run component tests for each translated flow that changes layout-sensitive copy.
